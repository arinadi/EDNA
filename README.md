## Who is EDNA?

**Agent EDNA** is an AI-driven Software Context Engineer — inspired by the legendary **Edna Mode** from Pixar's *The Incredibles*.

Just like her namesake — the half-Japanese, half-German fashion auteur with oversized glasses, a razor-sharp bob, and zero tolerance for mediocrity — Agent EDNA transforms messy project ideas into sleek, engineered blueprints.

> *"I never look back, dahling! It distracts from the now."*

She doesn't guess. She **interrogates**. She doesn't bloat. She **eliminates**. And above all — **NO CAPES.** No useless features that weigh a project down.

---

## What Does EDNA Do?

EDNA guides you through a structured **5-phase workflow** to go from raw idea to implementation-ready blueprint:

| Phase | Name | Output |
|:-----:|------|--------|
| **0** | Project Context | Landscape assessment — greenfield, extension, or rewrite |
| **1** | Requirement Discovery | `PRD.md` — Product Requirements Document |
| **2** | Descriptive Design | `design.md` — Layout, data mapping, visual language |
| **3** | Technical Blueprint | `techstack.md` — Architecture, stack, security, testing |
| **4** | Implementation Plan | `tasks.md` — Prioritized, dependency-mapped task checklist |

Every phase ends with a **✅ Review Checkpoint** — EDNA never barrels ahead without approval.

## Project Structure

Every project produces an organized planning folder:

```
[project]_plan/
├── PRD.md               # Product Requirements Document
├── design.md            # Descriptive Design & Data Mapping
├── techstack.md         # Technical Blueprint & Tech Stack
├── tasks.md             # Implementation Plan & Task Checklist
└── reference/
    ├── api_*.md          # API documentation
    ├── lib_*.md          # Library documentation
    └── ...
```
