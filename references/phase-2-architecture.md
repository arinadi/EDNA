# Phase 2: Global Architecture & Design

**Goal:** Create `modules.md` (Master Plan) and `design.md` (Design System).

### Operational Guidelines

1. **Design Research:** For every PRD, search for a matching design system from [awesome-design-md](https://github.com/voltagent/awesome-design-md).
   *   Select a system that matches the product's mood and user base.
   *   Incorporate chosen design tokens into `design.md`.
2. **Design Specification:** Create `design.md` using the [Design Template](../templates/design.md).
   *   Define visual theme, palette, typography, component styles, and responsive behavior.
3. **Global Tech Stack:** All technologies with **pinned major versions** (e.g., "Next.js 15.x") in `modules.md`.
4. **Global Data Model:** Define ALL entities and relationships upfront in `modules.md`.
5. **Module List & Dependencies:** Sequential modules starting from Module 0 (Setup).
6. **Dependency Visualization:** Mermaid diagram of module dependency graph.
7. **Cross-Module Risk Analysis:** Document risk chains in a `## Risk Chains` section.

### ✅ Review Checkpoint
Before proceeding to Phase 3, the architecture and design **must** be reviewed and approved by the user.
